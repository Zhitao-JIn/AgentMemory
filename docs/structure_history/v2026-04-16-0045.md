# 文件结构快照 - 配置冲突修复

**版本**: v2026-04-16-0045

## 变更说明

### 修复配置冲突

**问题**: `src/config.py` 和 `src/router/litellm_client.py` 使用不同的配置方式，导致冲突。

**解决方案**: 统一使用环境变量方式。

---

## 修改的文件

### `src/config.py`

**变更前**:
```python
# 硬编码的空字符串
qwen_key: str = ""
zhipu_key: str = ""
```

**变更后**:
```python
# 从环境变量读取
qwen_api_key: str = Field(default_factory=lambda: os.getenv("QWEN_API_KEY", ""))
zhipu_api_key: str = Field(default_factory=lambda: os.getenv("ZHIPU_API_KEY", ""))
```

**其他变更**:
- 默认提供商改为 `"qwen"`（更符合国内使用场景）
- 使用 `Field(default_factory=...)` 模式统一读取环境变量

## 配置统一方案

| 配置类型 | 来源 | 用途 |
|----------|------|------|
| API Keys | 环境变量 (`.env`) | 敏感信息，不提交到 Git |
| Base URLs | 环境变量 (`.env`) | 自定义请求地址 |
| 默认提供商 | `config.py` | 非敏感默认值 |
| 默认层级 | `config.py` | 非敏感默认值 |

## 使用方式

```bash
# 1. 创建 .env 文件
cp .env.example .env

# 2. 配置 API Key
echo "QWEN_API_KEY=sk-xxx" >> .env

# 3. 使用
python -c "from container import Container; c = Container(); print(c.chat('你好'))"
```
