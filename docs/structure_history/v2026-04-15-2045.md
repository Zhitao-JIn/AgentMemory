## 变更说明

### 问题

用户指出 `layer.py` 中存在非通过依赖注入实现的方法。之前接口文件被修改，将扩展方法（`update_context`, `get_context`, `handle_query_request`, `receive_feedback`, `broadcast_to_reasoning`, `archive_to_memory`, `query_memory`, `provide_feedback` 等）直接加入到基础接口 `IMemory`, `IPerception`, `IReasoning` 中，而不是通过继承接口 `IMemoryExtended` 等。

### 解决方案

1. **所有方法改为依赖注入**：layer.py 中的所有方法现在都优先使用注入的函数，没有注入时使用默认实现
2. **实现所有抽象方法**：确保所有接口中定义的抽象方法都有实现

### 修改的文件

#### `src/memory/layer.py`
- 添加 `update_context_fn`, `handle_query_fn`, `receive_feedback_fn` 注入参数
- `update_context()`: 优先使用注入函数，否则使用默认实现
- `handle_query_request()`: 优先使用注入函数，否则使用默认实现
- `receive_feedback()`: 优先使用注入函数，否则无操作
- `get_context()`: 新增实现，获取近期记忆摘要

#### `src/perception/layer.py`
- 添加 `receive_input_fn` 注入参数
- `receive_input()`: 优先使用注入函数，否则使用默认实现
- `broadcast_to_reasoning()`: 新增实现，发送数据给推理层
- `archive_to_memory()`: 新增实现，归档数据到记忆层

#### `src/reasoning/layer.py`
- 添加 `query_memory_fn`, `provide_feedback_fn`, `run_fn` 注入参数
- `query_memory()`: 优先使用注入函数，否则直接调用记忆层
- `provide_feedback()`: 优先使用注入函数，否则直接调用记忆层
- `run()`: 优先使用注入函数，否则使用默认的分析 + 检索 + 生成流程

#### `container.py`
- 导入 `receive_input` 函数并注入到 `PerceptionLayer`

# 所有方法现在遵循 DI 原则
class MemoryLayer(IMemory):
    def __init__(
        self,
        add_fn: Callable,
        search_fn: Callable,
        get_all_fn: Callable,
        update_context_fn: Optional[Callable] = None,
        handle_query_fn: Optional[Callable] = None,
        receive_feedback_fn: Optional[Callable] = None,
    ):
        self._add = add_fn
        self._search = search_fn
        self._get_all = get_all_fn
        self._update_context = update_context_fn  # 可选注入
        self._handle_query = handle_query_fn      # 可选注入
        self._receive_feedback = receive_feedback_fn  # 可选注入

    def update_context(self, data: Dict) -> Dict:
        if self._update_context:
            return self._update_context(data)
