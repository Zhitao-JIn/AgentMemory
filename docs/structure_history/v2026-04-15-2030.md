## 变更说明

### 删除的冗余文件

`src/protocols/` 目录在初始设计中用于层间通信协议，但后续发现：

1. **无外部引用**: 没有文件 `from src.protocols` 或 `import protocols`
2. **接口重复**: `protocols/interfaces.py` 定义的扩展接口与 `src/interfaces.py` 功能重叠
3. **消息结构未使用**: `protocols/message.py` 的 `Message` 数据结构未被实际使用

### 删除的文件列表

```
src/protocols/
├── __init__.py          (12 行)
├── interfaces.py        (116 行)
└── message.py           (22 行)
```

共计 3 个文件，150 行代码。

### 保留的设计

层间通信通过直接引用实现，而非通过消息总线：
- `PerceptionLayer` 直接持有 `_reasoning` 和 `_action` 引用
- `ReasoningLayer` 直接持有 `_memory` 引用
- 方法调用直接使用 Python 函数调用，而非消息传递

这种设计更简单、更高效，适合当前项目规模。
