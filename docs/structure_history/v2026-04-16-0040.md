### 新增 API 请求地址配置

支持通过环境变量自定义各提供商的 API 请求地址（BASE_URL）。

## 修改的文件

### `src/config.py`
- 新增各提供商的 `base_url` 配置项

### `src/router/litellm_client.py`
- 新增 `_setup_api_key_and_base_for_model()` 函数
- `call_litellm()` 支持 `api_base` 参数
- 自动从环境变量读取 `XXX_BASE_URL`

### `.env.example`
- 新增各提供商的 `BASE_URL` 配置示例（注释状态）

## 支持的环境变量

| 变量名 | 默认地址 | 说明 |
|--------|---------|------|
| `QWEN_BASE_URL` | https://dashscope.aliyuncs.com/compatible-mode/v1 | Qwen/通义千问 |
| `ZHIPU_BASE_URL` | https://open.bigmodel.cn/api/paas/v4 | 智谱 AI |
| `BAICHUAN_BASE_URL` | https://api.baichuan-ai.com/v1 | 百川 |
| `ANTHROPIC_BASE_URL` | https://api.anthropic.com | Anthropic |
| `OPENAI_BASE_URL` | https://api.openai.com/v1 | OpenAI |
| `GOOGLE_BASE_URL` | https://generativelanguage.googleapis.com | Google |

## 使用示例

### 使用代理访问 Qwen

```bash
# .env
QWEN_API_KEY=sk-xxx
QWEN_BASE_URL=https://your-proxy.com/v1
```

### 本地 Ollama 模型

```bash
# .env
OLLAMA_BASE_URL=http://localhost:11434/v1
```

```python
from container import Container
agent = Container()
response = agent.router.generate(prompt="你好", model="ollama/llama3.1:8b")
```
