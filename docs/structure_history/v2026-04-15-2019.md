## 变更说明

### DI 优化 - ReasoningLayer 依赖注入方式改进

#### 变更前
```python
# container.py
self.reasoning = ReasoningLayer(
    retrieve_fn=lambda ctx: retrieve(ctx, self.memory),  # 使用 lambda 包装
)
```

#### 变更后
```python
# container.py
self.reasoning = ReasoningLayer(
    retrieve_fn=retrieve,  # 直接注入函数
    memory=self.memory,    # 显式注入 IMemory
)

# src/reasoning/layer.py
def __init__(
    self,
    retrieve_fn: Callable[[Dict[str, Any], "IMemory"], Dict[str, Any]],
    memory: Optional["IMemory"] = None,  # 新增参数
):
    self._memory = memory
```

### 改进理由

1. **更清晰的依赖关系**: `ReasoningLayer` 明确声明需要 `IMemory` 依赖
2. **便于测试**: 可以独立传递 mock 的 `memory` 对象
3. **符合 DI 原则**: 避免在容器中使用 lambda 包装器
