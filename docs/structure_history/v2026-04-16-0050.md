# 文件结构快照 - 配置统一修复

**版本**: v2026-04-16-0050  
**时间**: 2026-04-16 00:50  

## 变更说明

### 修复配置读取问题

**问题**: `litellm_client.py` 直接从环境变量读取 API Key，而不是从 `config.py` 读取。

**解决方案**: 修改 `litellm_client.py` 优先从 `config.py` 读取配置。

---

## 修改的文件

### `src/router/litellm_client.py`

**变更前**:
```python
import os
result["api_key"] = os.getenv("QWEN_API_KEY")
```

**变更后**:
```python
from src.config import config

# 优先使用 config 中的配置
if CONFIG_AVAILABLE and config:
    result["api_key"] = config.qwen_api_key
    result["api_base"] = config.qwen_base_url
else:
    # 回退到环境变量
    result["api_key"] = os.getenv("QWEN_API_KEY")
```

---

## 配置优先级

```
1. 调用时传入的参数 (api_key, api_base)
       ↓
2. config.py 中的配置 (从 .env 读取)
       ↓
3. 环境变量 (如果 config 不可用)
```

---

## 配置流程

```
.env 文件
  ↓ (dotenv load_dotenv)
环境变量
  ↓ (config.py Field default_factory)
config 对象
  ↓ (litellm_client.py import config)
call_litellm()
  ↓
litellm_completion()
```
